   517→
   518→Follow the document type strategy defined in SOUL.md Rule 1.

   526→- **Default (all document types)**: ❗ **Absolutely NEVER force page breaks before H1/H2** — content flows naturally, do not insert `PageBreak()` before section headings. This is an iron rule.
   527→- Resume / contract / letter → No content page breaks
   528→- Short article → No content page breaks

   532→**Anti-tear (mandatory — but with height limit):**
   533→```python
   534→from reportlab.platypus import KeepTogether
   535→from reportlab.lib.pagesizes import A4
   536→from reportlab.lib.units import inch
   537→
   538→# Maximum height for KeepTogether blocks: 40% of page height
   539→# Blocks taller than this should NOT be kept together — they cause
   540→# the preceding page to be mostly empty when the block gets pushed down.
   541→MAX_KEEP_HEIGHT = A4[1] * 0.4  # ~336pt ≈ 12cm
   542→
   543→def safe_keep_together(elements):
   544→    """Wrap elements in KeepTogether only if their total height is reasonable.
   545→    If too tall, keep only the FIRST TWO elements together (e.g. title + table/image header)
   546→    to prevent title-content separation, while letting the rest paginate naturally.
   547→    """
   548→    total_h = 0
   549→    for el in elements:
   550→        w, h = el.wrap(A4[0] - 2*inch, A4[1])
   551→        total_h += h
   552→    if total_h <= MAX_KEEP_HEIGHT:
   553→        return [KeepTogether(elements)]
   554→    elif len(elements) >= 2:
   555→        # Keep at least title + first element together to prevent title orphaning
   556→        return [KeepTogether(elements[:2])] + list(elements[2:])
   557→    else:
   558→        return list(elements)
   559→
   560→# Heading stays with first paragraph
   561→story.extend(safe_keep_together([
   562→    heading_paragraph,
   563→    first_body_paragraph,
   564→]))
   565→
   566→# Image + caption: keep together ONLY if image is small
   567→story.extend(safe_keep_together([image, caption_paragraph]))
   568→
   569→# Table title + table: keep together ONLY if table is short
   570→if len(table_data) <= 15:
   571→    story.extend(safe_keep_together([table_title, table]))
   572→else:
   573→    # Long table: just keep title with the table start
   574→    story.extend(safe_keep_together([table_title]))
   575→    story.append(table)
   576→```
   577→
   578→**⚠️ KEY INSIGHT:** `KeepTogether` is the #1 cause of “empty page with chart on next page”.
   579→When a `KeepTogether` block is taller than the remaining space on the current page,
   580→ReportLab pushes the **entire block** to the next page, leaving the current page mostly empty.
   581→**Always use `safe_keep_together()` to cap the maximum block height.**
   582→
   583→**⚠️ Note:** Do NOT use `KeepWithNext` - it is unreliable in ReportLab 4.x.

   687→**Insertion script (single output PDF):**
   688→```python
   689→from pypdf import PdfReader, PdfWriter
   690→
   691→A4_W, A4_H = 595.28, 841.89  # A4 in points
   692→
   693→def normalize_page_to_a4(page):
   694→    """Scale a page to A4 if its dimensions don't match."""
   695→    box = page.mediabox
   696→    w, h = float(box.width), float(box.height)
   697→    if abs(w - A4_W) > 2 or abs(h - A4_H) > 2:
   698→        page.scale_to(A4_W, A4_H)
   699→    return page
   700→
   701→def insert_cover(cover_pdf, body_pdf, output_pdf):
   702→    """Insert cover as first page of body PDF → single output file."""
   703→    writer = PdfWriter()
   704→    # Cover as page 1
   705→    cover_page = PdfReader(cover_pdf).pages[0]
   706→    writer.add_page(normalize_page_to_a4(cover_page))
   707→    # Body pages follow
   708→    for page in PdfReader(body_pdf).pages:
   709→        writer.add_page(normalize_page_to_a4(page))
   710→    writer.add_metadata({'/Title': 'Report Title', '/Author': 'Z.ai', '/Creator': 'Z.ai'})
   711→    with open(output_pdf, 'wb') as f:
   712→        writer.write(f)
   713→```

You must call `normalize_page_to_a4()` to unify dimensions before insertion.

   904→### Table Width Management (⚠️ Prevent Table Overflow)
   905→
   906→**Most common table bug: columns overflow the right page margin because colWidths are hardcoded without checking available space.**
   907→
   908→**Iron Rule: The sum of all colWidths MUST NOT exceed `available_width`.**
   909→
   910→```python
   911→from reportlab.lib.pagesizes import A4
   912→from reportlab.lib.units import inch
   913→
   914→page_width = A4[0]  # 595.28pt
   915→left_margin = 1.0 * inch   # 72pt (adjust to your doc's actual margins)
   916→right_margin = 1.0 * inch  # 72pt
   917→available_width = page_width - left_margin - right_margin  # ≈ 451pt
   918→
   919→# ─── Method 1: Proportional widths (RECOMMENDED) ───
   920→# Define column ratios, auto-scale to fit available_width
   921→col_ratios = [0.25, 0.40, 0.20, 0.15]  # must sum to 1.0
   922→col_widths = [r * available_width for r in col_ratios]
   923→table = Table(data, colWidths=col_widths)
   924→
   925→# ─── Method 2: Fixed + flex columns ───
   926→# Some columns have fixed width (numbers, dates), rest fills remaining space
   927→fixed_cols = {0: 80, 3: 60}  # col 0 = 80pt, col 3 = 60pt
   928→fixed_total = sum(fixed_cols.values())
   929→flex_count = 4 - len(fixed_cols)  # total columns minus fixed
   930→flex_width = (available_width - fixed_total) / flex_count
   931→col_widths = []
   932→for i in range(4):
   933→    col_widths.append(fixed_cols.get(i, flex_width))
   934→table = Table(data, colWidths=col_widths)
   935→
   936→# ─── Method 3: Let ReportLab auto-calculate (simplest, less control) ───
   937→# Pass colWidths=None - ReportLab auto-sizes based on content
   938→# ⚠️ Risk: may still overflow if content is too wide
   939→table = Table(data)  # no colWidths
   940→```

   943→1. Calculate `available_width` from your actual page size and margins
   944→2. Sum all colWidths - must be ≤ `available_width`
   945→3. For CJK text: characters are wider than Latin - budget ~12pt per CJK char at 10pt font
   946→4. If a column has long text (descriptions, policies), use Paragraph() wrapping inside cells (not plain strings) - plain strings don't wrap and will overflow
   947→5. Test with the longest expected content in each column

  1008→## Auto-Generated Table of Contents
  1009→
  1010→### ❌ FORBIDDEN: Manual Table of Contents
  1011→**NEVER manually create TOC with hardcoded page numbers.**
  1012→
  1013→### ⚠️ MANDATORY: TOC requires a cover page
  1014→**Unless the user explicitly requests no cover, if a document has a Table of Contents, it MUST have a cover page.** Structure: Cover (page 1) → TOC (page 2) → Content (page 3+). Do not generate a TOC without a preceding cover page.
  1015→
  1016→### ✅ ALWAYS use auto-generated TOC:
  1017→
  1018→```python
  1019→from reportlab.platypus import SimpleDocTemplate, Paragraph, PageBreak, Spacer
  1020→from reportlab.platypus.tableofcontents import TableOfContents
  1021→from reportlab.lib.utils import simpleSplit
  1022→import hashlib
  1023→
  1024→class TocDocTemplate(SimpleDocTemplate):
  1025→    def afterFlowable(self, flowable):
  1026→        if hasattr(flowable, 'bookmark_name'):
  1027→            level = getattr(flowable, 'bookmark_level', 0)
  1028→            text = getattr(flowable, 'bookmark_text', '')
  1029→            key = getattr(flowable, 'bookmark_key', '')
  1030→            # MUST pass key as 4th element - without it, TOC entries won't have clickable links
  1031→            self.notify('TOCEntry', (level, text, self.page, key))
  1032→
  1033→doc = TocDocTemplate("document.pdf", pagesize=letter)
  1034→story = []
  1035→
  1036→toc = TableOfContents()
  1037→toc.levelStyles = [
  1038→    ParagraphStyle(name='TOCHeading1', fontSize=14, leftIndent=20, fontName='FreeSerif'),
  1039→    ParagraphStyle(name='TOCHeading2', fontSize=12, leftIndent=40, fontName='FreeSerif'),
  1040→]
  1041→story.append(Paragraph("<b>Table of Contents</b>", styles['Title']))
  1042→story.append(toc)
  1043→story.append(PageBreak())
  1044→
  1045→def add_heading(text, style, level=0):
  1046→    # Generate a unique bookmark key for this heading
  1047→    key = 'h_%s' % hashlib.md5(text.encode()).hexdigest()[:8]
  1048→    p = Paragraph('<a name="%s"/>%s' % (key, text), style)
  1049→    p.bookmark_name = text
  1050→    p.bookmark_level = level
  1051→    p.bookmark_text = text
  1052→    p.bookmark_key = key  # This key enables clickable TOC links
  1053→    return p
  1054→
  1055→# ⚠️ Orphan Heading Prevention (NOT a page break rule)
  1056→# If an H1 heading would appear in the bottom 15% of the page with no room
  1057→# for at least a few lines of content, push it to the next page.
  1058→# This is NOT "every H1 starts a new page" — it only triggers when the heading
  1059→# would be orphaned at the very bottom.
  1060→available_height = A4[1] - top_margin - bottom_margin
  1061→H1_ORPHAN_THRESHOLD = available_height * 0.15  # ~100pt ≈ 3.5cm — enough for heading + 2 lines
  1062→
  1063→def add_major_section(text, style):
  1064→    """Add an H1-level heading with orphan prevention (NOT forced page break)."""
  1065→    return [
  1066→        CondPageBreak(H1_ORPHAN_THRESHOLD),  # Only break if heading would be orphaned at page bottom
  1067→        add_heading(text, style, level=0),
  1068→    ]
  1069→
  1070→story.extend(add_major_section("Chapter 1: Introduction", styles['Heading1']))
  1071→# ... content ...
  1072→
  1073→doc.multiBuild(story)  # MUST use multiBuild for TOC
  1074→```
  1075→
  1076→**⚠️ CRITICAL TOC LINK RULES:**
  1077→1. `afterFlowable` MUST pass `key` as the 4th element in the notify tuple - without it, TOC entries have no clickable links
  1078→2. `add_heading` MUST set `bookmark_key` AND embed `<a name="key"/>` in the Paragraph text - this creates the link destination
  1079→3. The key must be unique per heading - use a hash of the heading text
